远程桌面

2018-12-23 2 min read # frp # 内网穿透 # 远程桌面 # rdpwrap

需要远程控制Windows，迫于teamviewer开始收费，识别为商业账号，连接5分钟就断开。换！

frp

很好用，用于内网穿透。
服务端：
基本不用修改配置，直接启动就可以了

frps.ini
[common]
bind_port = 7000

nohup ./frps -c frps.ini > frps.log 2>&1 &

客户端：
需要修改为服务端的ip地址，其他配置自己随便

frpc.ini
[common]
server_addr = 服务端的ip地址
server_port = 7000

[RDP]
type = tcp
local_ip = 127.0.0.1
local_port = 3389
remote_port = 6000

启动客户端

frpc -c frpc.ini

发现Win 10 家庭版的 没有远程桌面设置，无语微软。。。

RDPWrap

使用RDPWrap
以管理员身份执行install.bat就安装完了。
之后打开RDPConf.exe查看是否正常，如果没有listening，那么需要替换rdpwrap.ini文件

\Program Files\RDP Wrapper\rdpwrap.ini

替换后去服务里重启remote desktop service就正常了。
最终：
通过服务端IP地址:6000就可以远程桌面到内网的Windows机器啦~
